# DoodleRun iOS app

SwiftUI + MapKit. Talks to the FastAPI service in `../server/` to generate
animal-shaped running routes, previews them on a map, and exports as GPX.

## Requirements

- Xcode 15+ (uses `Map` content closure + `MapPolyline`, iOS 17 SDK)
- iOS 17+ deployment target
- The `server/` FastAPI service running and reachable from the simulator
  (default `http://localhost:8000`)

## Configuration

Default API base URL is `http://localhost:8000`. The simulator can hit the
host machine's localhost directly. To run on a physical device on the same
Wi-Fi, change the default in `RouteService.init` to your dev machine's
LAN IP (e.g. `http://192.168.1.42:8000`) â€” the Info.plist already permits
local-network HTTP.

## What's wired up

- `task { await loadShapes() }` populates the shape picker on launch from
  the server's `/shapes` endpoint, so adding a sixth animal on the backend
  appears in the iOS picker without any client-side changes.
- Tapping the map moves the green pin (the route center).
- "My Location" runs `CLLocationManager.requestLocation()` after the
  authorization prompt and centres on the result.
- "Generate" POSTs to `/generate`; on success the snapped polyline draws
  in blue and the idealized outline in red dashed.
- "Export GPX" presents `UIActivityViewController` so the user can save to
  Files, AirDrop, or open in Garmin Connect / Strava (both register the
  `.gpx` UTI).

## Known limitations

- No offline mode â€” without the FastAPI server reachable, all flows show
  an alert and the picker stays empty.
- ATS exceptions in Info.plist permit plaintext HTTP for localhost +
  local-network development. Switch the API to HTTPS and remove the
  exceptions before any production / TestFlight build.
- The `MapPolyline` rendering is iOS 17+ only. Lowering the deployment
  target requires a `UIViewRepresentable` wrapper around `MKMapView` in
  `MapView.swift`.
